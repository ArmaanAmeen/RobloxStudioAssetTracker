# Use Cases

## Project Title

Roblox Studio Project Asset Tracker

## Main Actor

Roblox Developer/Project Manager

## Use Case List

### 1. Add Asset

The user enters asset information such as name, type, Roblox asset ID, Studio path, status, and notes. The system saves the asset and displays it in the table.

### 2. View Assets

The user opens the application and can view all saved assets in a table.

### 3. Edit Asset

The user selects an existing asset, changes its information, and saves the updated version.

### 4. Delete Asset

The user selects an asset and removes it from the tracker.

### 5. Search Asset

The user types part of an asset name into the search box. The system displays matching assets.

### 6. Filter by Type

The user selects an asset type such as Weapon, Map, Sound, or Script. The system only displays assets with that type.

### 7. Filter by Status

The user selects a status such as Finished or Needs Fixing. The system only displays assets with that status.

### 8. Update Asset Status

The user changes the status of an asset to Finished, In Progress, Needs Fixing, or Not Started.

### 9. Add Notes

The user writes notes about an asset, such as bugs, missing parts, or design information.

### 10. Export Assets

The user clicks the export button. The system creates a CSV file containing the current asset list.
